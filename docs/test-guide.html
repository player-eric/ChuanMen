0. 快速开始 — 环境与登录

0.1 访问地址

0.2 测试账号

唯一可用的测试账号：cm@gmail.com
登录后身份为 Yuan（管理员 / 超级 Host），可访问所有功能包括管理后台。
其他邮箱登录会提示："当前仅支持测试账号 cm@gmail.com 登录"。

0.3 登录流程

  1. 首次打开网站 → 默认未登录状态
  2. 右上角点击 「登录」 按钮 → 进入登录页
  3. 页面展示蓝色提示框，标注测试账号 cm@gmail.com
  4. 输入邮箱 → 点登录 → 自动跳转首页
  5. 登录后右上角出现头像 + 「退出」 按钮

0.4 退出登录

右上角点击 「退出」 → 清除会话 → 回到未登录首页。

提示：所有数据为 Mock（硬编码），表单提交不会真正保存。刷新页面数据恢复原样。

12. 全局导航 & 响应式

12.1 移动端 (< 900px)

  • 顶部 AppBar：汉堡菜单 + 页面标题 + 暗色切换 + 登录/退出
  • 汉堡菜单 Drawer：用户信息、成员墙、关于串门儿、账号设置、管理后台(仅admin)、退出登录
  • 底部 5-Tab 导航栏（仅登录后）：动态、推荐、活动、感谢卡、我
  • 子页面显示返回箭头（替代汉堡图标）

12.2 桌面端 (≥ 900px)

  • 左侧 240px 永久 Drawer：品牌名 + 5 个主导航 + 成员墙/关于/设置/管理后台
  • 底部 Tab 栏隐藏
  • 内容区最大宽度 1100px，居中

12.3 导航一致性检查

  • 所有子页面返回按钮能正确回退
  • 页面标题与当前路由一致（如 /events → 「活动」，/profile → 「我的页面」）
  • 浏览器标题同步更新（格式：串门儿 - XXX）

13. 深色模式

  • 右上角太阳/月亮图标切换
  • 所有页面在深色模式下可读性正常
  • 卡片、表格、弹窗背景正确适配
  • 管理后台也支持深色模式切换

14. 已知限制 & 注意事项

项目说明
数据持久化所有数据为前端 Mock，刷新后重置。仅登录状态存储在 localStorage。
表单提交按钮可点击但不会真正保存数据（无后端连接）。
图片使用 Unsplash 占位图，加载速度受网络影响。
测试账号仅 cm@gmail.com 可登录，其他邮箱会报错。
管理后台编辑/删除/发送等操作有 UI 反馈但无实际后端逻辑。
SSR vs CSRAmplify 部署为纯 CSR（客户端渲染），首屏加载可能一瞬白屏。本地 npm run dev 为 SSR 模式。
推荐详情/discover/:category/:id 页面为通用模板，内容较简单。
